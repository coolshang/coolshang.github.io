git常用指令

  • git
  view

提示：所有的操作都在gitBash里

###1.git init
首先我们先创建一个目录并进入该目录。为了避免遇到各种莫名其妙的问题，请确保目录名（包括父目录）不包含中文。

mkdir test
cd test

然后我们就可以进行下一步了:

git init

通过这一步可以将这个目录变成Git可以管理的仓库。这一步会生成一个.git目录，这个目录是来跟踪管理版本库的，没事千万不要手动修改这个目录里的文件，如果乱改后，就把git仓库给破坏了。
在Windows下禁止使用记事本编辑或者新建任何文本文件。因为记事本的格式不是utf-8。可以用编辑器比如Notepad++、submit、webstorm等一些编辑软件。
我们可以新建一个文本learnGit.txt，内容如下:

Git is a version control system.
Git is free software.

###2.git add
然后我们可以使用git add的方法将我们刚才新建的文件learnGit.txt添加到仓库:

git add learnGit.txt

执行后，没有任何显示，就表示添加成功。

###3.git commit
使用git commit告诉Git,把文件提交到仓库:

git commit -m "wrote a readme file"

git commit命令,-m后面输入的是本次提交的说明,可以输入任意内容,当然最好是有意义的,这样你就可以从历史记录里方便地找到改动记录。
如果你嫌麻烦不想输入-m "xxx"，可以不输入，但是强烈不建议你这么做,因为输入说明对自己对别人阅读都很重要。

###4.git status
我们已经成功地添加并提交了一个learnGit.txt文件，现在我们继续修改这个文件，内容如下:

Git is a distributed version control system.
Git is free software.

现在，运行git status命令看看结果:

git status

git status命令可以让我们时刻掌握仓库当前的状态，上面的命令告诉我们，readme.txt被修改过了，但还没有准备提交的修改。

###5.git diff
使用git diff这个命令可以快速的找到上次你修改了什么。
git diff就是查看difference,显示的格式正是Unix通用的diff格式。通过这个命令输出看到，我们在第一个添加了一个”distributed”单词。

###6.git log
使用git log命令显示从最近到最远的提交日志。
如果嫌输出信息太多,看得眼花撩乱的,可以试试加上--pretty=oneline参数。
你看到一大串数字与字母结合的是commit id(版本号),和svn不一样，Git的commit id不是1,2,3……递增的数字,而是一个SHA1计算出来的一个非常大的数字，用十六进制表示,而且你看到的commit id和我的肯定不一样,以你自己的为准。如果使用可视化工具查看Git历史,就可以更清楚地看到提交历史的时间线。

###7.git reset
使用git reset可以将我们的文件learnGit.txt回退到某个你想要的版本。
首先，Git必须知道当前版本是哪个版本,在Git中,用HEAD表示当前版本，也就是当前最新的提交。
上个版本就是HEAD^,上上个版本就是HEAD^^,当然往上100个版本写100个^比较数不过，所以写成HEAD~100。
